Stage 1, Lesson 2: HTML markup nad documents

HTML markup is made of tags /NAME/CONTENTS//NAME/
Instead of "/" the tags in coding are written with ">" signs. One before and expanding, and one after and closing. The content is surrounded by an opening and a closing tag. Each tag has a name that the sotware recognizes. A slash is always in front of the name of the closing tag. Some tags are void tags so they don`t require a closing tag. The entire line can be called an element.

HTML tags are various and provide a vast array of options for users.
You can create invisible boxes to organise texts, add links, contain additional contents and so on. The HTML document consists of html tag surrounding the whole document, the head which has meta-data, the title and the body of the html. Among the most important tags are "/span/ >" and "/div/". Both contain elements but the first one does it inline while the other one always has a separate line. Thus a "div" is a block level, meaning it's on its own separate line while a span is inline, so it's a child of another block level element.

Stage 2, Lesson 1: Creating a Structured Document with HTML

  1. Developer Tools (in the Browser)

    HTML elements are either inline or block. Block elements form an invisible box around the content inside of them. Thus they contain the content and can be used to manipulate it in many ways.

  2. The "tree-like structure" of HTML

    When programmers talk about the "DOM" (document object model) they are talking about the the tree-like structure of a page. The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. These relationships can be interpreted as a tree like structure which it actually forms though in a complex manner. All elements are connected in one way or another, if nothing else all elements within this document are sons and daughters of the /body/ element which encompasses all of them.

  3. The relationship between indented HTML and boxes

    This structure both shows the specific positions of elements and contents and can be used to better manipulate them. When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

  4. Text Editors (for programming)

    When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Stage 2, Lesson 2: CSS

  1. What is CSS?

    CSS stands for Cascading Style Sheets. CSS is code written to control the "style" of HTML elements. It can change how elements look, their position, color, font and may other things.

  2. How does CSS "control" HTML?

    If you want to write CSS that makes all h1 elements have a black background and white text, you would write:
    h1 {
    background-color : black;
    color: white;
    }
    In this example, the h1 is a selector. It says to the browser "I want you to apply the rules I'm about to tell you to every h1 element." After the h1 there is left a curly brace { and at the bottom there is a matching right curly brace }. Everything between these curly braces will be interpreted as a "rule" that should be applied to every h1 element. The line of code that says background-color:black; is a declaration. bacground-color is a property and black is the value of that property.

  3. Selecting by class

    If we only want to add style to certain h1s, we can use class selectors.

  4. Three ways of including CSS styling in a web page:

    1. Write CSS in the /head/ of your html.
      This is good for small projects. At the top of your html add a /head/ element. Inside the /head/ element add a /style/ element. Write your CSS element and then put the html for the rest of your page below.

    2. Link your html to a separate CSS file
      Write all of your structural html in one file (ex. main.html). In a separate file (ex. main.css) put all of your CSS code. Add a /head/ element to the top of your html document. Add a /link/ tag inside the head element (keep in mind link is a void tag!). Finally, add the following attributes to your /link/ : *rel="stylesheet", *type="text/css", *href="main.css"

    3. Write your style inline with html
       With this method you modify the style atributes of every individual element.

2.Avoiding Repetition

Avoiding repetition is important for a variety of reasons. The first is avoiding errors: when programmers don't avoid repetition, they will often have to do the same thing over and over and make mistakes. This may mean copying and pasting certain style attributes into many HTML elements or rewriting the same code with just minor differences many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears they will have issues of various degree. Another reason is consistency: Instead of manually giving each segment of codes the same style, each of them can be assigned the same "div" and the same "class" and then specify the style for that class in only one place! By avoiding repetion, it's then very easy for to make changes to style.
